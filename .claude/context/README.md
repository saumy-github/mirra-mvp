# Context Directory

> [!IMPORTANT]
> **AI should NOT auto-update this file or other files in this directory unless explicitly instructed to do so.**

This directory contains project context, business requirements, and domain knowledge that helps guide development decisions.

## Purpose

The `context/` folder answers:

- **What** are we building?
- **Why** are we building it?
- **Who** is it for?
- **What** are the goals and constraints?

## File Types

This directory can contain various document formats:

- Markdown files (`.md`) - Easy to read and edit
- PDF files (`.pdf`) - For formal requirements documents
- Text files (`.txt`) - For simple notes
- Any other document format

## Suggested Files

Consider creating the following files to provide comprehensive context:

### Core Files

- **`project-overview.md`** - High-level description of the project
- **`requirements.md`** - Functional and non-functional requirements
- **`target-audience.md`** - User personas and target market
- **`business-goals.md`** - Business objectives and success metrics

### Optional Files

- **`domain-knowledge.md`** - Industry-specific knowledge needed
- **`use-cases.md`** - Key user scenarios and workflows
- **`constraints.md`** - Technical, budget, or timeline constraints
- **`stakeholders.md`** - Key decision makers and their priorities

## When to Update

- **Before starting**: Establish initial context
- **During discussion phase**: Clarify requirements
- **As project evolves**: Update when goals or scope change

## How AI Uses This

During the **Discussion** and **Planning** phases, I'll reference these documents to:

- Understand the problem space
- Make informed architectural decisions
- Suggest features aligned with business goals
- Ask relevant clarifying questions
