Step 3 Locked Decisions

Step 3 is VTO.

Architecture:
- new user-facing app layer goes in `vto/`
- `clo_workspace` stays the REST/plugin/integration layer only

Critical Step 3 dependency model:
- Step 3 is the first stage that depends on both earlier steps together
- it consumes Step 1 avatar output
- it consumes Step 2 panel DXF output

Current known implementation fact:
- the current 11-step CLO automation is actually orchestrated by `clo_workspace/plugins/clo_automation_steps/pipeline.py`
- current thin entrypoint is `clo_workspace/plugins/clo_automation_client.py`

Locked Step 3 canonical runner:
- `vto/run_vto.py`

Locked Step 3 input-selection behavior:
- no separate `vto/input/` folder for now
- `run_vto.py` should work like an interactive runner similar to Step 1
- prompt for `user_id` first
- resolve latest Step 1 run for that user by default
- allow overriding the Step 1 run number
- then prompt for `cloth_id` and `size_id`
- resolve latest Step 2 run for that `cloth_id + size_id` pair by default
- allow overriding the Step 2 run number
- print fully resolved Step 1 and Step 2 source runs
- ask the user to confirm before starting the CLO flow

Locked Step 3 output naming:
- format: `<avatar_run>__<product_run>__<vto_run_number>`
- example: `u_001-001__c_001-s_001-001__001`
- VTO run number increments per selected avatar-run + product-run combination

Locked Step 3 minimal run structure:

`vto/output/u_001-001__c_001-s_001-001__001/`
- `input.json`
- `run_summary.json`

Step 3 run-folder content rules:
- do not duplicate Step 1 or Step 2 source artifacts into the VTO run folder by default
- record and consume them by reference
- `input.json` should record selected source run IDs, resolved source paths, and run-selection metadata
- `run_summary.json` should record Step 3 execution status, step-by-step results, and any output/export references
- CLO-side exports/projects may still remain under `clo_workspace/exports/` and `clo_workspace/projects/` during transition, but their paths should be recorded in `run_summary.json`
- automatic export/save is not required for this cleanup
- if export/save remains manual, `run_summary.json` should say manual/skipped rather than pretending artifacts were created
