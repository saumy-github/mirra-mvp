# Coding Rules Directory

> [!IMPORTANT]
> **AI should NOT auto-update this file or other files in this directory unless explicitly instructed to do so.**

This directory contains technical standards, conventions, and architectural guidelines that must be followed during development.

## Purpose

The `coding-rules/` folder answers:

- **How** should code be written?
- **What** technologies should be used?
- **How** should files be organized?
- **What** patterns and conventions should be followed?

## Core Files

### Required Files

Create these essential files to establish your coding standards:

1. **`tech-stack.md`** - Frontend/backend technologies and versions
2. **`file-structure.md`** - Project organization and folder hierarchy
3. **`coding-style.md`** - Formatting, naming conventions, and code patterns

### Recommended Files

## When to Update

- **Project setup**: Define initial standards
- **Planning phase**: Reference when designing solutions
- **Execution phase**: Strictly follow these rules
- **As needed**: Update when adopting new patterns or technologies

## How AI Uses This

During **Planning** and **Execution** phases, I'll:

- Follow the exact tech stack specified
- Adhere to file structure conventions
- Apply coding style rules to all code
- Implement consistent patterns across the codebase
- Flag any conflicts between requirements and rules

## Critical Rules

Any rules marked as **CRITICAL** or **MUST** in these files take precedence over AI defaults and will be strictly followed without exception.
